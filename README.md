# Book Review System API Documentation

This documentation provides a comprehensive guide for installing, configuring, running, and testing the Book Review System API built using Django and Django REST Framework with JSON Web Tokens JWT for secure authentication and distinct permission tiers for user and administration management. The key features include secure registration, JWT-based login, token refresh mechanics, protected password updates, a publicly readable book catalog with write operations strictly limited to administrators, and a review system where authenticated users can write comments and assign ratings while safe object-level checks ensure users can modify or delete only their own records. To run the project locally, clone the repository, access the root directory, initialize a dedicated virtual environment using python -m venv venv and activate it, then install all required libraries using pip install -r requirements.txt before executing the schema migrations sequence with python manage.py migrate and generating an administrative superuser account with python manage.py createsuperuser to finally boot up the local development web server using python manage.py runserver where the application endpoints will be active and listening at http://127.0.0.1:8000/.

To handle authentication via Postman, users pass their credentials via a POST request to /api/token/ which returns an access token string along with a refresh token, then for every subsequent request to an authenticated endpoint, users copy the access token string, navigate to the Authorization tab, select Bearer Token as the type parameter, and paste the access token into the input area. The full reference for the system endpoints consists of thirteen managed routes starting with POST /api/register/ to create standard user accounts using a payload containing username and password fields, POST /api/token/ to validate user records and return the JWT pair, POST /api/token/refresh/ to renew an expired access token using a valid refresh token, and POST /api/change-password/ for authenticated users to upgrade their current credentials after verifying identity by submitting old_password and new_password parameters.

The book catalog management includes GET /api/books/ to return a structured array listing all saved books records, GET /api/books/<id>/ to return specific metadata fields of a target book matching the provided ID parameter, POST /api/books/ for admin users only to commit a new book asset to the database by sending a JSON payload with title, author, and description, PUT /api/books/<id>/ for admin users only to overwrite data parameters of an existing book instance, and DELETE /api/books/<id>/ for admin users only to completely remove a target book from the database while returning a status of 204 No Content upon success. The review processing system handles GET /api/books/<book_id>/reviews/ to list all reviews belonging to a specific book target, POST /api/books/<book_id>/reviews/ for authenticated users to register a rating integer and a comment string linked directly to a verified book, PUT /api/reviews/<id>/ for object owners only to edit their own review comment and score, and DELETE /api/reviews/<id>/ for object owners only to purge their designated review directly from the active system database while returning a status of 204 No Content upon completion. All data validation layers are handled through serializers for comprehensive inbound data sanitization and outbound JSON serialization, enforcing standard HTTP status codes including 201 Created for successful records generation, 204 No Content for successful deletions, 401 Unauthorized for missing or expired tokens, 403 Forbidden for restricted administrative or non-owner resource access, and 404 Not Found for non-existent resource records.
